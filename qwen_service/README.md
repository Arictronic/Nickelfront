# Qwen Service

**Полностью автономный** мини-сервис для работы с Qwen API с REST интерфейсом и поддержкой **авто-продолжения** ответов.

## Особенности

- ✅ **Полная автономность** — не зависит от основного проекта (AI.Qwen, Domain.chat, Core и т.д.)
- ✅ **REST API** — простой HTTP интерфейс для интеграции
- ✅ **Авто-продолжение** — интеллектуальное определение и продолжение незавершённых ответов
- ✅ **Keep-alive логирование** — стабильная работа при долгих запросах
- ✅ **Защита API** — поддержка токенов для защиты endpoints

## Быстрый старт

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

**Требуемые пакеты:**
- `fastapi>=0.109.0` — веб-фреймворк
- `uvicorn[standard]>=0.27.0` — ASGI сервер
- `pydantic>=2.5.0` — валидация данных
- `requests>=2.31.0` — HTTP клиент (для Qwen API)
- `httpx>=0.26.0` — HTTP клиент (для client.py)

### 2. Настройка

Откройте `config.json` и настройте параметры:

```json
{
  "host": "127.0.0.1",
  "port": 8767,
  "token": "ваш_qwen_токен",
  "api_key": "ключ_для_защиты_api",
  "model": "qwen-coder",
  "thinking_enabled": true,
  "search_enabled": true,
  "auto_continue_enabled": true,
  "max_continues": 5
}
```

### 3. Запуск сервиса

```bash
python service.py
```

### 4. Тестирование

В новом терминале:

```bash
# Запуск тестов
python client.py

# Интерактивный чат
python client.py --chat
```

## Авто-продолжение ответов

Сервис поддерживает **автоматическое продолжение** ответов с интеллектуальным определением необходимости продолжения.

### Как это работает:

1. **Автоматическое определение** — сервис анализирует ответ и определяет, завершён ли он:
   - Проверяет флаг `can_continue` от API
   - Анализирует окончание ответа (обрыв на середине предложения, многоточие, и т.д.)
   - Проверяет незакрытые скобки, кавычки, блоки кода
   - Определяет незавершённые списки

2. **Keep-alive логирование** — при долгой генерации сервис логирует прогресс каждые 15 секунд, чтобы соединение не обрывалось

3. **Таймауты** — увеличенные таймауты (120 сек) для обработки долгих ответов

4. Процесс повторяется пока ответ не будет завершён или не будет достигнут лимит `max_continues`

### Признаки незавершённого ответа:

- Ответ обрывается на середине слова/предложения
- Есть незакрытые скобки `(`, кавычки `"`, блоки кода ` ``` `
- Ответ заканчивается на маркер списка (`1.`, `•`, `-`)
- Ответ заканчивается на союз или вводное слово («и», «или», «например»)
- Явный флаг `can_continue: true` от API

### Настройка:

```json
{
  "auto_continue_enabled": true,  // Включить авто-продолжение
  "max_continues": 5              // Максимальное количество продолжений
}
```

### Через API:

```bash
# Получить настройки
curl http://127.0.0.1:8767/config/auto_continue

# Установить настройки
curl -X POST "http://127.0.0.1:8767/config/auto_continue?enabled=true&max_continues=3" \
  -H "Authorization: Bearer qwen-service-key-2026"
```

### Переопределение для конкретного запроса:

```python
# Отключить авто-продолжение для одного запроса
response = httpx.post(
    "http://127.0.0.1:8767/messages",
    headers=headers,
    json={
        "session_id": session_id,
        "message": "Привет!",
        "auto_continue": False  # Переопределение глобальной настройки
    }
)
```

### Ответ с информацией об авто-продолжении:

```json
{
  "session_id": "...",
  "response": "...",
  "thinking": "...",
  "auto_continue_performed": true,  // Было ли выполнено авто-продолжение
  "continue_count": 2,              // Количество продолжений
  "can_continue": false,            // Можно ли ещё продолжить
  "last_message_id": 42
}
```
