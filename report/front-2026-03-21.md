## Что сделано (актуально на 2026-03-26)
- Полностью заменён фронтовый “мок” по данным: интерфейс переподключён к реальным endpoint-ам бэка по **papers** (`/api/v1/papers/...`).
- Обновлены ключевые маршруты и страницы:
  - `/` — landing, метрики берутся из БД
  - `/dashboard` — запуск парсинга и KPI по количеству статей
  - `/papers` — каталог статей (list + фильтры/сортировка на клиенте + пагинация)
  - `/papers/:id` — карточка статьи + разбиение `full_text/abstract` на части по настройке токенов
  - `/papers/:id/report` — отдельная страница отчета по частям
  - `/vector-search` — поиск (fallback через `POST /api/v1/papers/search`)
  - `/jobs` — статус парсинга (эвристика по росту `papers/count`, так как отдельного endpoint celery-status по `task_id` нет)
  - `/database` — отображение фактического состояния по доступным endpoints
- Удалены остатки старого слоя “патентов”:
  - `src/utils/mockData.ts`, `src/store/patentStore.ts`
  - старые компоненты и типы для `/patents`

## Авторизация (важно)
- Формы `login/register` в текущем проекте работают только как UI-логика.
- В backend не найдено endpoint-ов auth, поэтому настоящий доступ к данным через БД/токены пока невозможно подключить без изменения backend.
- Где менять для реального auth:
  - `frontend/src/api/auth.ts`
  - `frontend/src/hooks/useAuth.ts`
  - `frontend/src/store/authStore.ts`
  - и добавить endpoint-ы в backend.

## Эндпоинты, с которыми работает фронт
- `GET    /api/v1/papers?limit&offset&source`
- `GET    /api/v1/papers/count?source`
- `GET    /api/v1/papers/id/{id}`
- `POST   /api/v1/papers/search`
- `POST   /api/v1/papers/parse`
- `POST   /api/v1/papers/parse-all`
- `DELETE /api/v1/papers/id/{id}`
